## 一、文档概述

### 1.1 文档目的

本产品需求文档旨在详细描述闪电帐PRO系统的功能优化需求，基于与竞品"精品功能"系统的对比分析，明确需要新增和优化的功能模块，为产品设计、开发和测试团队提供清晰的需求指导。

### 1.2 项目背景

闪电帐PRO是一款面向电商商家的财务管理系统，目前已具备经营概览、财务核算、资金管理、库存成本、经营分析、费用中心、税务管理以及完整的出纳管理等核心功能。通过与市场竞品的功能对比分析，发现在订单管理、成本配置、单据中心等方面存在功能缺失，需要进行补充和优化以提升产品竞争力。

### 1.3 目标用户

闪电帐PRO的目标用户主要包括以下群体：

| 用户角色 | 用户特征 | 核心需求 |
|---------|---------|---------|
| 电商店主 | 经营1-5家店铺的中小商家 | 快速了解经营状况，简化财务管理 |
| 财务人员 | 负责多店铺财务核算的专职人员 | 精准核算成本利润，高效对账 |
| 出纳人员 | 负责资金收付和流水管理的人员 | 管理资金流水，处理对账差异 |
| 运营人员 | 负责店铺日常运营的人员 | 分析销售数据，优化运营策略 |

### 1.4 术语定义

| 术语 | 定义 |
|-----|------|
| SKU | Stock Keeping Unit，库存量单位，用于区分商品的最小单位 |
| 商家编码 | 商家自定义的商品唯一标识码 |
| 特殊订单 | 不按常规方式计入财务统计的订单，如样品、员工内购等 |
| 成本波动 | 商品成本价格在一定时期内的变化幅度 |
| 对账差异 | 系统记录与平台账单之间的金额差异 |

## 二、功能需求概述

### 2.1 功能模块总览

本次功能优化共涉及8个功能模块，按优先级分为三个等级：

| 优先级 | 功能模块 | 预估工期 | 业务价值 |
|-------|---------|---------|---------|
| P0（高） | 订单管理模块 | 3-4天 | 核心业务功能，直接影响用户日常操作效率 |

| P0（高） | 成本配置中心 | 2-3天 | 精准成本管理，提升利润核算准确度 |
| P1（中） | 地区分析增强 | 1-2天 | 辅助运营决策，优化区域营销策略 |
| P1（中） | 单据中心 | 2-3天 | 完善业务闭环，提升仓储发货效率 |
| P1（中） | 多平台数据同步 | 1-2天 | 提升数据时效性，减少人工操作 |
| P2（低） | 帮助中心 | 1天 | 降低用户学习成本，减少客服压力 |

```
订单管理模块

```

## 三、详细功能需求

### 3.1 订单管理模块（P0）

#### 3.1.1 功能概述

订单管理模块是电商财务系统的核心功能之一，用于集中展示和管理来自各电商平台的订单数据。该模块需要支持订单的查询、筛选、详情查看、批量操作等功能，帮助用户高效管理日常订单业务。

#### 3.1.2 功能结构

订单管理模块包含以下子功能：

| 子功能 | 功能说明 | 优先级 |
|-------|---------|-------|
| 订单列表 | 展示所有订单的基本信息，支持分页和排序 | 必须 |
| 多维度筛选 | 按状态、来源、时间、金额等条件筛选订单 | 必须 |
| 订单详情 | 展示单个订单的完整信息，包括财务数据 | 必须 |
| 批量操作 | 支持批量打印、标记、导出等操作 | 必须 |
| 订单搜索 | 按订单号、买家信息等关键词搜索 | 必须 |

#### 3.1.3 订单列表

**页面布局**

订单列表页面采用左侧筛选面板 + 右侧主表格的布局方式，筛选面板默认收起，点击"筛选"按钮展开。

**数据字段**

订单列表表格需展示以下字段：

| 字段名称 | 字段说明 | 数据类型 | 是否必显 |
|---------|---------|---------|---------|
| 选择框 | 用于批量操作的复选框 | Checkbox | 是 |
| 订单编号 | 平台订单唯一标识 | String | 是 |
| 下单时间 | 订单创建时间 | DateTime | 是 |
| 买家信息 | 买家昵称（脱敏显示） | String | 是 |
| 商品信息 | 商品名称和SKU摘要 | String | 是 |
| 订单金额 | 订单实付金额 | Decimal | 是 |
| 平台/店铺 | 订单来源平台和店铺名称 | String | 是 |
| 订单状态 | 当前订单状态 | Enum | 是 |
| 操作 | 查看详情、标记等操作按钮 | Button | 是 |

**订单状态定义**

| 状态值 | 状态名称 | 状态颜色 | 说明 |
|-------|---------|---------|------|
| pending_payment | 待付款 | 灰色 | 订单已创建，等待买家付款 |
| pending_shipment | 待发货 | 橙色 | 买家已付款，等待商家发货 |
| shipped | 已发货 | 蓝色 | 商家已发货，物流运输中 |
| completed | 已完成 | 绿色 | 买家已确认收货，交易完成 |
| refunding | 退款中 | 红色 | 买家申请退款，处理中 |
| refunded | 已退款 | 灰色 | 退款已完成 |
| closed | 已关闭 | 灰色 | 订单已取消或关闭 |

**分页规则**

默认每页显示20条订单记录，支持切换为50条或100条。分页组件需显示当前页码、总页数、总记录数，支持跳转到指定页。

#### 3.1.4 多维度筛选

**筛选条件**

筛选面板需支持以下筛选维度：

| 筛选维度 | 筛选类型 | 筛选选项 | 默认值 |
|---------|---------|---------|-------|
| 订单状态 | 多选复选框 | 待付款、待发货、已发货、已完成、退款中、已退款、已关闭 | 全选 |
| 订单来源 | 多选复选框 | 抖音、淘宝、京东、拼多多、快手、小红书 | 全选 |
| 时间范围 | 日期选择器 | 开始日期 - 结束日期 | 近7天 |
| 订单金额 | 范围滑块 | 最小金额 - 最大金额 | 0 - 不限 |
| 特殊订单 | 单选按钮 | 全部、仅普通订单、仅特殊订单 | 全部 |

**筛选交互**

用户修改筛选条件后，点击"应用筛选"按钮触发数据刷新；点击"重置"按钮恢复所有筛选条件为默认值。筛选面板支持收起/展开切换，收起时在顶部显示当前生效的筛选条件标签。

#### 3.1.5 订单详情

**详情面板**

订单详情采用右侧滑出面板的形式展示，点击订单行或"查看详情"按钮触发。详情面板宽度为400px，包含以下信息区块：

**订单基本信息**

| 信息项 | 说明 |
|-------|------|
| 订单编号 | 平台订单号，支持一键复制 |
| 下单时间 | 订单创建时间，精确到秒 |
| 付款时间 | 买家付款时间 |
| 买家昵称 | 脱敏显示，如"张**" |
| 收货地区 | 省份+城市，如"浙江杭州" |
| 收货地址 | 完整收货地址（需权限） |
| 买家备注 | 买家下单时的备注信息 |
| 商家备注 | 商家添加的内部备注 |

**商品明细**

| 信息项 | 说明 |
|-------|------|
| 商品图片 | 商品主图缩略图 |
| 商品名称 | 商品标题，超长截断显示 |
| SKU信息 | 规格信息，如"颜色:黑色 尺码:M" |
| 商品数量 | 购买数量 |
| 商品单价 | 商品售价 |
| 商品总额 | 单价 × 数量 |
| 运费 | 订单运费 |
| 优惠金额 | 各类优惠总额 |
| 实付金额 | 买家实际支付金额 |

**财务信息**

| 信息项 | 说明 | 计算公式 |
|-------|------|---------|
| 商品成本 | 商品采购成本总额 | Σ(SKU成本 × 数量) |
| 平台扣点 | 平台技术服务费 | 实付金额 × 扣点比例 |
| 运费险 | 运费险费用 | 固定金额或按订单计算 |
| 达人佣金 | 达人带货佣金 | 实付金额 × 佣金比例 |
| 其他费用 | 其他扣费项目 | 根据实际情况 |
| 预估毛利 | 预估订单毛利润 | 实付金额 - 成本 - 各项费用 |
| 毛利率 | 毛利润占比 | 预估毛利 / 实付金额 × 100% |

**操作按钮**

详情面板底部需提供以下操作按钮：

| 按钮名称 | 按钮类型 | 功能说明 |
|---------|---------|---------|
| 打印配货单 | 主按钮 | 生成并打印该订单的配货单 |
| 标记特殊订单 | 次按钮 | 将订单标记为特殊订单 |
| 导出子订单 | 次按钮 | 导出该订单的详细数据 |
| 添加备注 | 次按钮 | 为订单添加商家内部备注 |

#### 3.1.6 批量操作

**批量选择**

表格表头提供全选复选框，勾选后选中当前页所有订单。选中订单后，顶部操作栏显示已选数量和批量操作按钮。

**批量操作功能**

| 操作名称 | 功能说明 | 触发条件 |
|---------|---------|---------|
| 批量打印 | 批量生成并打印配货单 | 选中至少1条订单 |
| 批量标记 | 批量将订单标记为特殊订单 | 选中至少1条订单 |
| 批量导出 | 将选中订单导出为Excel文件 | 选中至少1条订单 |
| 全部导出 | 导出当前筛选条件下的所有订单 | 无需选中 |

#### 3.1.7 接口需求

**订单列表接口**

```
GET /api/orders
```

请求参数：

| 参数名 | 类型 | 必填 | 说明 |
|-------|-----|-----|------|
| page | Integer | 否 | 页码，默认1 |
| pageSize | Integer | 否 | 每页条数，默认20 |
| status | String[] | 否 | 订单状态数组 |
| platform | String[] | 否 | 平台来源数组 |
| startDate | Date | 否 | 开始日期 |
| endDate | Date | 否 | 结束日期 |
| minAmount | Decimal | 否 | 最小金额 |
| maxAmount | Decimal | 否 | 最大金额 |
| keyword | String | 否 | 搜索关键词 |
| isSpecial | Boolean | 否 | 是否特殊订单 |

响应数据：

| 字段名 | 类型 | 说明 |
|-------|-----|------|
| total | Integer | 总记录数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |
| list | Order[] | 订单列表 |

**订单详情接口**

```
GET /api/orders/:orderId
```

响应数据包含订单基本信息、商品明细、财务信息等完整数据。

### 3.2 特殊订单标记功能（P0）

#### 3.2.1 功能概述

特殊订单标记功能用于将不按常规方式计入财务统计的订单进行标记，如样品订单、员工内购、售后补发等。标记后的订单将按照特定规则参与或排除出财务统计，确保财务数据的准确性。

#### 3.2.2 标记类型定义

| 标记类型 | 类型编码 | 财务处理规则 | 适用场景 |
|---------|---------|-------------|---------|
| 样品订单 | sample | 不计入销售额和利润统计 | 寄送给达人、媒体的样品 |
| 员工内购 | staff_purchase | 按成本价计算，不计入正常利润 | 员工福利购买 |
| 售后补发 | after_sale | 不计入销售额，成本计入售后成本 | 质量问题补发商品 |
| 赠品订单 | gift | 不计入销售额，成本计入营销费用 | 活动赠品发放 |
| 其他 | other | 根据备注自定义处理 | 其他特殊情况 |

#### 3.2.3 标记弹窗设计

**弹窗触发**

用户可通过以下方式触发标记弹窗：单个订单的"标记特殊订单"按钮、批量选中后的"批量标记"按钮。

**弹窗内容**

| 区域 | 内容说明 |
|-----|---------|
| 受影响订单 | 显示受影响的订单数量和预估影响销售额 |
| 标记原因 | 单选按钮组，选择标记类型 |
| 其他原因输入 | 当选择"其他"时显示，输入自定义原因 |
| 生效时间 | 单选按钮：立即生效 / 指定时间 |
| 时间选择器 | 当选择"指定时间"时显示 |
| 备注说明 | 可选的文本输入框，补充说明 |

**操作按钮**

| 按钮 | 功能 |
|-----|------|
| 确认标记 | 提交标记请求，成功后关闭弹窗并刷新列表 |
| 取消 | 关闭弹窗，不做任何操作 |

#### 3.2.4 标记状态显示

已标记的特殊订单在列表中需有明显的视觉标识：订单行左侧显示彩色标签，标签颜色和文字根据标记类型区分；鼠标悬停标签时显示标记详情（标记人、标记时间、标记原因）。

#### 3.2.5 取消标记

已标记的订单支持取消标记操作，取消后订单恢复为普通订单，重新参与正常的财务统计。取消标记需要记录操作日志，包括操作人、操作时间、取消原因。

### 3.3 成本配置中心（P0）

#### 3.3.1 功能概述

成本配置中心用于管理商品的成本价格，支持按商家编码或SKU配置成本，支持多时段成本配置和成本波动预警。准确的成本数据是利润核算的基础，该功能对财务数据准确性至关重要。

#### 3.3.2 功能结构

| 子功能 | 功能说明 | 优先级 |
|-------|---------|-------|
| 成本列表 | 展示所有商品的成本配置 | 必须 |
| 成本编辑 | 单个或批量修改成本价格 | 必须 |
| 批量导入 | 通过Excel批量导入成本数据 | 必须 |
| 成本历史 | 查看成本变更历史记录 | 必须 |
| 多时段配置 | 配置不同时间段的成本价格 | 应有 |
| 成本预警 | 成本波动超阈值时告警 | 应有 |

#### 3.3.3 成本列表

**列表字段**

| 字段名称 | 字段说明 | 数据类型 |
|---------|---------|---------|
| 商家编码 | 商品唯一标识码 | String |
| 商品名称 | 商品标题 | String |
| SKU | 规格信息 | String |
| 当前成本价 | 当前生效的成本价格 | Decimal |
| 新成本价 | 待修改的成本价格（可编辑） | Decimal |
| 生效时间 | 新成本价的生效时间 | DateTime |
| 最后更新 | 成本最后更新时间 | DateTime |
| 操作 | 保存、历史记录等按钮 | Button |

**筛选条件**

| 筛选项 | 类型 | 说明 |
|-------|-----|------|
| 店铺选择 | 下拉单选 | 筛选指定店铺的商品 |
| 商家编码 | 文本输入 | 按商家编码搜索 |
| 商品名称 | 文本输入 | 按商品名称模糊搜索 |
| 成本状态 | 下拉单选 | 全部/已配置/未配置 |

#### 3.3.4 成本编辑

**单个编辑**

在成本列表中，用户可直接在"新成本价"列输入新的成本价格，选择生效时间后点击保存按钮完成修改。

**生效时间选项**

| 选项 | 说明 |
|-----|------|
| 立即生效 | 保存后立即使用新成本价 |
| 指定时间 | 选择未来某个时间点生效 |
| 下月1日 | 从下月1日0点开始生效 |

**批量编辑**

支持选中多条记录后批量设置相同的成本调整比例或固定金额。

#### 3.3.5 批量导入

**导入模板**

系统提供标准的Excel导入模板，模板包含以下列：

| 列名 | 必填 | 说明 |
|-----|-----|------|
| 商家编码 | 是 | 商品唯一标识 |
| SKU编码 | 否 | SKU标识，为空则应用到所有SKU |
| 成本价格 | 是 | 新的成本价格 |
| 生效时间 | 否 | 为空则立即生效 |

**导入流程**

用户下载模板后填写数据，上传Excel文件，系统进行数据校验。校验通过后显示预览，用户确认后执行导入。导入完成后显示成功/失败数量和详细的错误信息。

#### 3.3.6 成本历史

点击"历史记录"按钮可查看该商品的成本变更历史，包括：变更时间、变更前成本、变更后成本、变更原因、操作人。

#### 3.3.7 多时段成本配置

**功能说明**

支持为同一商品配置多个时间段的不同成本价格，适用于采购成本随时间变化的场景。

**配置规则**

| 规则 | 说明 |
|-----|------|
| 时间段不重叠 | 同一商品的多个成本配置时间段不能重叠 |
| 自动切换 | 系统根据订单时间自动匹配对应时段的成本 |
| 默认成本 | 未配置时段使用默认成本价 |

#### 3.3.8 成本波动预警

**预警设置**

| 设置项 | 说明 | 默认值 |
|-------|------|-------|
| 上涨预警阈值 | 成本上涨超过该比例时告警 | 10% |
| 下降预警阈值 | 成本下降超过该比例时告警 | 15% |
| 预警通知方式 | 系统消息/邮件/短信 | 系统消息 |
| 预警接收人 | 接收预警通知的用户 | 当前用户 |

**预警触发**

当用户修改成本价格时，系统自动计算变动比例。如果变动比例超过预警阈值，则在保存前弹出确认提示，并在保存后发送预警通知。

#### 3.3.9 全店铺应用

勾选"全店铺应用此配置"后，当前成本配置将自动同步到所有店铺中相同商家编码的商品上，避免多店铺重复配置。

### 3.4 地区分析增强（P1）

#### 3.4.1 功能概述

在现有经营分析模块中增加地区分析功能，通过地图热力图和排行榜的形式展示各地区的销售数据，帮助商家了解区域销售分布，优化区域营销策略。

#### 3.4.2 功能组件

**地区热力地图**

在页面左侧展示中国地图，根据各省份的销售额数据着色显示热度。颜色从浅到深表示销售额从低到高，鼠标悬停省份时显示详细数据（销售额、订单数、占比）。

**省份销售排行**

在页面右侧展示省份销售排行榜，按销售额降序排列。每行显示：排名、省份名称、销售额、环比变化。支持切换排序维度（销售额/订单数/客单价）。

**城市下钻**

点击地图上的省份或排行榜中的省份，可下钻查看该省份内各城市的销售数据。

#### 3.4.3 数据筛选

| 筛选项 | 类型 | 说明 |
|-------|-----|------|
| 时间范围 | 日期选择器 | 筛选指定时间段的数据 |
| 平台筛选 | 多选下拉 | 筛选指定平台的数据 |
| 店铺筛选 | 多选下拉 | 筛选指定店铺的数据 |
| 商品类目 | 多选下拉 | 筛选指定类目的数据 |

#### 3.4.4 数据导出

支持将地区分析数据导出为Excel文件，包含：省份/城市、销售额、订单数、客单价、占比、环比变化等字段。

### 3.5 单据中心（P1）

#### 3.5.1 功能概述

单据中心用于管理电商业务中的各类单据，包括配货单、出库单、销售单、入库单等。通过单据管理实现业务流程的规范化和可追溯性。

#### 3.5.2 单据类型

| 单据类型 | 类型编码 | 用途说明 |
|---------|---------|---------|
| 配货单 | picking | 仓库拣货配货使用 |
| 出库单 | outbound | 记录商品出库信息 |
| 销售单 | sales | 记录销售交易信息 |
| 入库单 | inbound | 记录商品入库信息 |
| 退货单 | return | 记录退货入库信息 |

#### 3.5.3 配货单管理

**配货单生成**

配货单可通过以下方式生成：从订单管理页面单个或批量生成、按发货批次自动生成、手动创建配货单。

**配货单内容**

| 信息区域 | 包含字段 |
|---------|---------|
| 单据信息 | 配货单号、生成时间、打印状态 |
| 订单信息 | 关联订单号、买家信息、收货地址 |
| 商品明细 | 商品名称、SKU、数量、货位 |
| 配货状态 | 待配货/配货中/已配货/已发货 |

**配货单打印**

支持单个打印和批量打印，打印模板可自定义。打印内容包括：单据条码、商品信息、收货地址等。

#### 3.5.4 出库单管理

出库单记录商品从仓库发出的信息，与配货单关联。出库单包含：出库单号、出库时间、关联配货单、商品明细、物流信息。

#### 3.5.5 入库单管理

入库单记录商品入库信息，支持采购入库和退货入库两种类型。入库单包含：入库单号、入库时间、入库类型、供应商、商品明细、入库数量。

### 3.6 多平台数据同步（P1）

#### 3.6.1 功能概述

多平台数据同步功能用于从各电商平台拉取最新的订单、商品、财务等数据，确保系统数据的时效性和完整性。

#### 3.6.2 支持平台

| 平台名称 | 平台编码 | 同步内容 |
|---------|---------|---------|
| 抖音电商 | douyin | 订单、商品、结算、达人数据 |
| 淘宝/天猫 | taobao | 订单、商品、结算数据 |
| 京东 | jd | 订单、商品、结算数据 |
| 拼多多 | pdd | 订单、商品、结算数据 |
| 快手电商 | kuaishou | 订单、商品、结算数据 |
| 小红书 | xiaohongshu | 订单、商品、结算数据 |

#### 3.6.3 同步状态显示

在经营概览页面显示各平台的同步状态卡片，每个卡片包含：平台Logo和名称、最近同步时间、同步状态（正常/同步中/失败）、今日数据摘要（销售额、订单数）。

**状态定义**

| 状态 | 图标颜色 | 说明 |
|-----|---------|------|
| 正常 | 绿色 | 最近同步成功，数据已更新 |
| 同步中 | 蓝色 | 正在同步数据 |
| 失败 | 红色 | 最近同步失败，需要处理 |
| 未授权 | 灰色 | 平台未授权，无法同步 |

#### 3.6.4 同步操作

**手动同步**

用户可点击"一键同步"按钮触发所有已授权平台的数据同步，也可点击单个平台卡片的同步按钮只同步该平台。

**自动同步**

系统支持配置自动同步策略：同步频率（每小时/每2小时/每天）、同步时间段（避开业务高峰期）、同步失败重试次数。

#### 3.6.5 同步日志

系统记录每次同步的详细日志，包括：同步时间、同步平台、同步数据量、同步耗时、同步结果、错误信息（如有）。
